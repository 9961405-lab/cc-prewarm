cc-prewarm

让 Claude Code / Codex 的 5 小时额度窗口在高峰中间重置

高峰时段可能横跨两个窗口,缓解额度紧张

一图看懂原理

不用 cc-prewarm 高峰整段挤在一个窗口里
用 cc-prewarm(06:00 自动预热) 高峰横跨两个窗口

一行安装

npm install -g cc-prewarm
cc-prewarm            # 启动 3 步设置向导

需要 Node.js 18 或以上

向导做哪三件事

扫描使用习惯

分别读 Claude Code 和 Codex 本地记录,画 24 小时分布图

推荐触发时间

为每个工具单独推荐:在你高峰前 3 小时触发

安装定时任务

写入系统定时任务,以后每天自动跑,你不用管

常用命令

cc-prewarm status
窗口状态 + 最近一次预热结果 + 7 天成功率
cc-prewarm doctor
体检 node / claude / codex / 定时任务 / 唤醒
cc-prewarm trigger
立刻发一次预热(失败会显示真实报错)
cc-prewarm logs
查看最近触发日志(带时间戳、彩色)
cc-prewarm history
最近触发记录 + 失败原因聚合
cc-prewarm uninstall
不想用了,一键移除定时任务

macOS:让 Mac 准点醒(只对清晨触发的人重要)

launchd 不会叫醒睡眠中的 Mac。如果你最早一次触发在凌晨/清晨(白天机器本来就醒着,这一步不用做),按"最早触发时间减 2 分钟"设置自动唤醒:

sudo pmset repeat wakeorpoweron MTWRFSU HH:MM:00

举例:

  • 触发在 06:00 → 改成 05:58:00
  • 触发在 08:00 → 改成 07:58:00
  • 触发在 14:00 → 不用设(下午机器肯定醒着)

不知道自己设了几点?跑 cc-prewarm doctor 看。
